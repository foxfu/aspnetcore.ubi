Uplay Web Test Report by Max Fu

  1. The project is basically keep the original structure to keep the test simple
  2. Entry point after build http://localhost:56653
  3. New added or updated folder and files
    • wwwroot/css
    • wwwroot/js
    • wwwroot/app.js
    • wwwroot/index.html
    • Controllers
  4. Use angularJs(1.7.2) as the framework to perform the data binding between webapi and client side
  5. Use bootstrap(4.1.2) library for the UI styling
  6. The 3rd party library(angularJs and bootstrap) already included in the project folder
  7. Implemented web apis
    • HttpGet: /api/games?userAccountId=xxxx, Get current or specific user owned games
    • HttpGet: /api/games/ownership?userAccountId=xxxx, Get specific user owned games and status, for admin use only
    • HttpGet: /api/games/all?userAccountId=xxxx, Get all games info with specific user, including those not owned by the user, for admin use only
    • HttpPost: /api/games/grant, Grant a game to user without key, for admin use only
    • HttpPost: /api/games/redeem, Redeem a game with key
    • HttpPost: /api/games/revoke, Revoke a game with specific, for admin use only
    • HttpPost: /api/login, User login
    • HttpPost: /api/usres, List all users, admin use only
    • HttpPost: /api/usres/update, Update user profile
  8. All client side logic is in app.js and included in one body controller to simplify the design
  9. Datebase.cs has been updated to singleton design pattern so all controllers can use the same data store in memory
  10. The index.html contains several part of the UI, some of them are shared by the normal user and admin like user profile part, this is also for simplify the design, different view controlled by scope flag variable
